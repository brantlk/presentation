Reviewing - Overall

Try it out. Say what you did.

Don't take on too many reviews.

Pro Tip: Look at the code coverage report.

Reviewing - Scoring

-1 : Needs work / bad idea

+1 : No problems

+0 : If you have a question, not willing to +1

+2 : No problems (core reviewer)

-2 : This is a bad idea (core reviewer)

Workflow+1 : Starts merging if Verify +1

Workflow-1 : Work in progress

Verify+1 : Testing says it's ok

Verify-1 : Testing failed

Submitting a change

Prepare the submission from the perspective of a reviewer.

Use WIP (workflow -1) to push something you're working on.

Commit messages

Think of it as advertising why someone should look at this change (vs all the other ones).

Commit message format

title

Answer to question: Why was the old behavior incorrect?

Answer to question: Why is the new behavior correct?

If you think a summary will help, go ahead.

References: bugs, work items, docs, etc.
